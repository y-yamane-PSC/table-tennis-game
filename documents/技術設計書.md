# 技術設計書：どきどきピンポン

## 1. アーキテクチャ概要

### 1.1 全体構成

- **フレームワーク**: React 19
- **言語**: TypeScript
- **描画**: HTML5 Canvas API
- **スタイリング**: CSS Modules / Tailwind CSS
- **ビルドツール**: Vite
- **状態管理**: React Context API + useState/useReducer

### 1.2 アーキテクチャパターン

- **コンポーネント指向**: 機能ごとにコンポーネントを分割
- **カスタムフック**: ゲームロジックをフックに分離
- **Canvas描画**: ゲームループをrequestAnimationFrameで実装
- **状態管理**: 画面状態はContext API、ゲーム状態はuseReducerで管理

- バウンド後の速度は維持

### 6.4 スマッシュ発動ロジック

```typescript
function canSmash(
  ball: Ball,
  racket: Racket,
  maxDistance: number = 100
): boolean {
  const distance = Math.sqrt(
    Math.pow(ball.x - racket.x, 2) + 
    Math.pow(ball.y - racket.y, 2)
  );
  return distance <= maxDistance;
}
```

## 13. 技術的考慮事項

### 13.1 Canvas描画の注意点

- **座標系**: 左上が(0, 0)
- **描画順序**: 背景 → 卓球台 → ラケット → ボール → パーティクル
- **クリア**: 毎フレーム全体をクリアして再描画

### 13.2 状態更新のタイミング

- **ゲーム状態**: useReducerで管理、ゲームループ内で更新
- **画面状態**: Context APIで管理、画面遷移時に更新
- **描画状態**: Canvas描画はゲームループ内で実行

### 13.3 イベントハンドリング

- **キーボードイベント**: useEffectでwindow.addEventListener
- **マウスイベント**: Canvas上でmousemove, clickを検知
- **クリーンアップ**: コンポーネントのアンマウント時にイベントリスナーを削除
