# 要件定義書：ミラクル・ラリー！キャンディ・マジック✨

## 1. プロジェクト概要

- **名称**: ミラクル・ラリー！キャンディ・マジック✨
- **コンセプト**: 「かんたん・かわいい・楽しい」をテーマにした、女児向け卓球ゲーム。
- **ターゲット**: 小学校低学年〜中学年の女児
- **主要技術**: React 19、TypeScript、Canvas API

---

## 3. 機能要件

### 3.1 画面機能

- **ホーム画面**
  - 難易度選択ボタン（ひらがな表記：やさしい／ふつう／むずかしい）
  - キラキラしたアニメーション付きのスタートボタン

- **ラケット選択画面**
  - 自分好みのラケットを選択可能
  - ラケットによって特殊効果がある

- **ゲーム画面**
  - キャラクター操作（左右方向キー）※キャラクターは左右にのみ移動可能
  - キャラクターが右方向に進む際はラケットは右持ち、左方向に進むと左持ちになる
  - ラケット操作（クリックまたはスペースキー）※ラケットはクリックまたはスペースキーで打ち返すことができ、ラケット自体を上下左右に直接操作することはできない
  - ホーム画面やラケット選択画面の操作（方向キーまたはマウス操作、確定操作はマウスクリックまたはスペース/エンターキー）
  - リアルタイムスコア表示（数字＋ハートなどのアイコン）
  - ヒット時のエフェクト（星やキラキラのパーティクル）

- **リザルト画面**
  - 勝敗結果の大きなイラスト表示（「かち」「まけ」）
  - 「もういちど あそぶ」ボタン
  - 「はじめに もどる」ボタン

### 3.2 ゲームロジック

- **難易度制御**
  - やさしい: ボール速度 0.7x / ラケットサイズ 1.5x / CPU追従性 低
  - ふつう: ボール速度 1.0x / ラケットサイズ 1.0x / CPU追従性 中
  - むずかしい: ボール速度 1.3x / ラケットサイズ 0.8x / CPU追従性 高

- **衝突判定**
  - ラケット表面での反射処理
  - 上下の壁面でのバウンド処理

- **ラケットとボールの設定**
  - ラケットには基本ステータスとして下記を設定する
    ・当たり判定（ラケットの幅・高さ）
    ・移動速度（キャラクターの左右移動速度）
    ・スマッシュ速度（スマッシュ時のボール速度倍率）

  - ラケットは下記の3種類を用意し、試合前に選択する
    ・ノーマルラケット：バランスの良い性能（基準値：当たり判定1.0、移動速度1.0、スマッシュ速度1.0）
    ・スピードラケット：スマッシュの速度が1.2倍になるが当たり判定が20%低下（移動速度はそのまま）
    ・ワイドラケット：当たり判定が1.2倍になるが移動速度が20%低下（スマッシュ速度はそのまま）

  - ボールはラリーが続くことで変化する
  - ボールが変化するタイミングはラリーが5の倍数のときとする（5ラリー、10ラリー、15ラリー...）

  - 変化するボールの種類は下記の五つの中からランダムとする
    ・イチゴ：一時的に（2ラリーの間）自分の当たり判定を1.2倍に広くする
    ・ハート：本物のボールを含め三つのハートを相手のコートに放つ（ボールが分身する）
    ・星：スマッシュ速度が1.5倍になるスマッシュを打つことができる
    ・キャンディ：一時的に（2ラリーの間）相手の当たり判定を0.8倍に狭くする
    ・リボン：左右に変化する球を放つことができる

  - 上記のボールの変化の継続時間は1ラリーとする（ボールが変化した状態で自分と相手が一回ずつ打ったら終了）
    ※イチゴとキャンディは例外として2ラリーの間効果が継続
  - ハートのボールは一点から三方向（中央0度、左斜め-30度、右斜め+30度）にボールが分身して放たれる（本物のボールのみ当たり判定あり）
  - 偽物のハートを打ち返すとハートがはじける演出あり
  - ハートのボールが分裂した際、相手のコートバウンド時に本物だけ「キラリ」と光る演出あり
  - リボンのボールは相手のコートにバウンドした際に、左右斜め45度のいずれかにランダムでバウンドする変化球
  - キャンディーのボールを相手が打ち返すと「ボールがうちにくくなったよ！」という文字を画面中央に表示する
  - イチゴのボールを打つことで適用される当たり判定の上昇は打ったプレイヤーにのみ適用（両方がイチゴのボールを打ち返したら両方に当たり判定上昇の効果が適用される）
  - イチゴのボールを自分が打ち返すと「ボールがうちやすくなったよ！」と画面中央に表示される

- **スマッシュの設定**
  - スマッシュはクリックまたはスペースキーで発動する
  - スマッシュが発動可能な条件：ボールがラケットから一定距離以内（例：100px以内）にある時
  - 通常のスマッシュ速度はボールの基本速度の1.5倍とする
  - 星ボールのスマッシュ速度はボールの基本速度の2.25倍（1.5倍×1.5倍）とする

### 3.3 ゲームルール

- **スコアのルール**
  - 11点先取で勝利とする
  - 最大ラリー数の制限や時間による制限はなし

- **ラリーのルール**
  - ボールが往復した回数をラリーとする（1往復 == 1ラリー）
  - ラリー数はボールがプレイヤーとCPUの間を往復するたびにカウントされる

- **ゲーム開始・終了のルール**
  - ゲーム開始時、ボールはプレイヤー側からサーブされる
  - 得点が入った後、ボールは得点を取った側からサーブされる
  - 11点先取でゲーム終了、リザルト画面に遷移する

## 4. デザイン要件（UI/UX）

- **配色**: パステルピンク、ラベンダー、ミントグリーンを基調とする
- **フォント**: 角の丸いフォント（Zen Maru Gothic 等）を使用、すべて「ひらがな・カタカナ」で表記
- **演出**
  - 得点時に「やったね！」などのポジティブなフィードバックを表示
    ・フィードバックは、得点時に「やったね！」「すごい！」等の演出を表示させる
    ・勝利時には紙吹雪とともに「よくがんばったね！」等の誉め言葉を表示
  - ヒット時のパーティクルは星やハートを用いて行う（星やハートが全方位に散らばるようなイメージ）
  - ボールがイチゴやハートの形に変化するなどの視覚的楽しみあり
  - 音響（ヒット音、得点音、背景音など）
    ・使用する音源ファイルは未定（最後に決める）
  - **表示**
    - ゲーム画面で表示されるのは卓球台、お互いのラケット、ボール、得点版のみとする（その他は風景として表示）
    - キャラクターは表示しない（ラケットのみ表示）

## 5. 非機能要件

- **レスポンシブ**: PCでプレイ可能
- **操作性**: 少ないキーで完結するシンプルなインターフェース
  - ゲーム画面での操作：左右方向キー（キャラクター移動）、クリックまたはスペースキー（ラケット操作）
- **アクセシビリティ**: ボタンを大きく配置し、押し間違いを防ぐ
- **画面遷移の流れ**: ホーム→難易度選択→ラケット選択→ゲーム画面→リザルト
  - リザルト後の遷移先（「もういちど あそぶ」はゲーム画面へ、「はじめに もどる」はホームへ遷移）
- **キャンバスサイズ**: 1280×720pxのHD

## 6. デプロイ・ホスティング

- **ホスティングサービス**: Vercel（推奨）
  - Vite + Reactアプリケーションに最適化されている
  - 無料プランで十分な機能を提供
  - GitHubリポジトリと連携して自動デプロイが可能
  - 高速なCDN配信
  - カスタムドメイン対応（無料）

- **デプロイ方法**
  - **方法1: GitHub連携（推奨）**
    1. GitHubリポジトリにコードをプッシュ
    2. VercelのダッシュボードからGitHubリポジトリを接続
    3. 自動的にビルドとデプロイが実行される
    4. 以降、mainブランチへのプッシュで自動デプロイ
  
  - **方法2: Vercel CLI**
    ```bash
    # Vercel CLIをインストール
    npm i -g vercel
    
    # プロジェクトで実行
    vercel
    ```

- **ビルド設定**
  - **ビルドコマンド**: `npm run build`
  - **出力ディレクトリ**: `dist`
  - **フレームワークプリセット**: Vite
  - **環境変数**: 不要（静的サイト）

- **代替ホスティングサービス**
  - **Netlify**: 静的サイト向け、無料プランあり、ドラッグ&ドロップデプロイ対応
  - **GitHub Pages**: 無料、GitHubと連携しやすい
  - **Cloudflare Pages**: 高速、無料プランあり

- **カスタムドメイン**
  - Vercelの無料プランでもカスタムドメインの設定が可能
  - DNS設定が必要
